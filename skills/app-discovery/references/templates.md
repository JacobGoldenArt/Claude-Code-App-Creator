This project follows a structured feature-by-feature development workflow. Before starting any work:

4. Run the dev server and verify existing features still work
5. Work on ONE feature at a time until all its tests pass
6. Update progress.md, commit, then end the session

## Usage Notes

### Status Indicators

Use these consistently across features:
- ðŸ”´ **Not Started** - No work begun
- ðŸŸ¡ **In Progress** - Currently being implemented
- ðŸŸ¢ **Complete** - All tests passing

### Feature Ordering

Order features by dependency:
1. Foundation features (no dependencies) first
2. Features that depend on #1 next
3. Continue building up the dependency tree
4. Nice-to-have features at the end

### Test Writing Guidelines

Each feature should have 8-15 tests covering:
- Basic visibility and presence
- Core functionality (happy path)
- User interactions
- State changes
- Edge cases
- Error handling

### Dependency Notation

- `None` - No dependencies, can be built first
- `Feature 1` - Depends on a single feature
- `Features 1, 3` - Depends on multiple features
